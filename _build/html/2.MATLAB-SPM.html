Installing MATLAB and SPM#

As mentioned in the previous section, SPM is the software we will be using to manipulate brain images on this course. SPM is developed by the Wellcome Centre for Human Neuroimaging at University College London (UCL) and is notable for being written in the MATLAB programming language. This means that we need a copy of MATLAB in order for SPM to run. As such, our first job is to get MATLAB up and running.

MATLAB#

MATLAB (MATrix LABoratory) is both a programming language and a scientific computing environment, developed by the MathWorks corporation. It was originally developed as a simple interactive matrix calculator, but has since expanded into a fully-fledged programming language with suites of functions for image processing, computer vision, statistics, machine learning and the development of graphical user interfaces (to name a few). In this section, we will see how to install MATLAB and will have a quick tour of the interface. We will then start learning how to use MATLAB in the next section.

Installing MATLAB#

Despite its roots in academia, MATLAB is no longer free software. If you are using a University cluster PC, MATLAB will already be installed and ready for you to use. However, if you want to use your own computer, you will need to install MATLAB with the University’s campus-wide license. To do this, first visit the MathWorks website and click the Get MATLAB button. You will be prompted to create an account. Make sure you do this using your University email address, otherwise the license will not be available. Once you have signed in, click the Install MATLAB button. Once you are on the downloads page, you can select the MATLAB version. For Windows, just download the latest version. If you are using a Mac, make sure you download version R2023a, otherwise there will be compatibility issues with SPM12.

SPM#

Now that MATLAB is installed, we can turn our attention to SPM. SPM is one of the oldest and most widely used software packages for processing brain images. It was originally developed in the early 90s by Karl Friston, as a means of processing and analysing PET data. With the advant of fMRI, the software was soon expanded for use with functional imaging data, as well as structural MRI, MEG and EEG datasets. The version of SPM we will be using is SPM12, which was released at the end of 2012 and is the 5th major version of the SPM software. SPM is a hugely import piece of academic software. The SPM team have been highly influential in the field of fMRI data analysis, developing many of the methods and approaches we will be covering on this course. For those interested in learning more, the history of SPM has been documented in the retrospective by Ashburner (2012).

Installing SPM#

At its most basic, SPM is a collection of MATLAB functions. You download the software as a folder, tell MATLAB where the folder is located and then launch the software by typing spm at the MATLAB prompt. The SPM launcher will then open, as shown below.

spm

In order to install SPM, follow along with the video further down this page. You will see how to download the software, get it working with MATLAB and finally how to use SPM to view an example image. You can download the example image from here, and can play around with it using the viewer below. Note that if you are using macOS, there will be some additional steps to perform to get SPM working, which are explained in the section below the video.

Resolving Errors on macOS#

For anyone installing SPM on macOS, there are some additional steps that will be required before the software will function. If you try and launch SPM, you will get errors pertaining to the bundled MEX files. The issue is that the macOS security system is rather over-zealous and will have quarantined the MEX files on the off-chance that they contain malicious code. In order to resolve this, we have to un-quarantine those files. Unfortunately, Apple do not want to make it too easy to bypass their security systems, so the only way to do this is to type some commands into the macOS Terminal.

To begin with, you need to open the Terminal application. It can be found inside Applications > Utilities, or you can press CMD + SPACE to bring up Spotlight search, and then type terminal. Once open, you need to move the current working directory of the terminal to the spm12 folder. Assuming you installed SPM inside Documents > MATLAB, you can use the cd command to move there by typing

cd ~/Documents/MATLAB/spm12

and hitting Enter. Then, you need to type (or copy + paste) the following command

find . -name "*.mexmaci64" -exec xattr -d com.apple.quarantine {} \;

and hit Enter. This command will work through the spm12 directory and un-quarantine all the macOS-specific MEX files. Once it has finished, close the Terminal and then re-open MATLAB and try launching SPM. With any luck, the errors about MEX files will have disappeared.

Resolving Other Problems#

If you hit any other problems when trying to get SPM to function, one of the first places to look is the SPM Wiki. This is a very useful technical resource with guides on many different aspects of SPM, including installation and custom compliation of the MEX files (which hopefully you never have to do!). Another resource is the SPM JISCM@il. This is a question-and-answer forum that both the users and developers of SPM frequent to discuss the software. If you are having issues, there is a good chance that someone else may have asked a similar question already on the list. So try searching the archives or creating a new post. If you still cannot resolve the problem on your own, please post on the relevant Blackboard discussion board for the current module. One of the teaching team, or even your fellow students, can respond to try and help.
